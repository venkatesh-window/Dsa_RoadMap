Sorting Algorithms Guide with LeetCode Problems

Introduction to Sorting

What is Sorting?

Arranging the elements of a list (or an array) in a specific order (ascending or descending).

Why Learn Different Sorting Algorithms?

  • Different algorithms have different time and space complexities
  • Some perform better on small datasets, others on large ones
  • Some are stable, others are not (important for sorting by multiple keys)
  • Understanding them improves problem-solving and algorithmic thinking

Key Terms:

  • Time Complexity: How the runtime grows with the input size
  • Space Complexity: The extra memory an algorithm requires
  • In-place Algorithm: Uses constant extra space (i.e., sorts within the original array)
  • Stable Algorithm: If two elements have the same value, their relative order from the original list is preserved

Part 1: Basic O(n²) Sorting Algorithms

These are simple to understand but less efficient for large datasets.

1. Selection Sort

Core Idea: Repeatedly find the minimum element (for ascending order) from the unsorted part and put it at the beginning.

Complexity Analysis:

  • Time Complexity: O(n²) in all cases
  • Space Complexity: O(1)
  • Stability: No

LeetCode Problems:

  • 75. Sort Colors - Medium
  • 215. Kth Largest Element in an Array - Medium

2. Bubble Sort

Core Idea: Repeatedly swap adjacent elements if they are in the wrong order. This causes the largest elements to "bubble up" to their correct position in each pass.

Complexity Analysis:

  • Time Complexity: O(n²) worst/average, O(n) best case (optimized)
  • Space Complexity: O(1)
  • Stability: Yes

LeetCode Problems:

  • 283. Move Zeroes - Easy
  • 905. Sort Array By Parity - Easy

3. Insertion Sort

Core Idea: Build the sorted array one item at a time by taking an element from the unsorted part and inserting it into its correct position in the sorted part.

Complexity Analysis:

  • Time Complexity: O(n²) worst/average, O(n) best case
  • Space Complexity: O(1)
  • Stability: Yes

LeetCode Problems:

  • 147. Insertion Sort List - Medium
  • 88. Merge Sorted Array - Easy

Part 2: Efficient O(n log n) & Recursive Algorithms

These are more efficient for large datasets and are based on the "Divide and Conquer" paradigm.

4. Merge Sort

Core Idea: Divide the array into two halves, sort each half recursively, and then merge the two sorted halves back together.

Complexity Analysis:

  • Time Complexity: O(n log n) in all cases
  • Space Complexity: O(n)
  • Stability: Yes

LeetCode Problems:

  • 912. Sort an Array - Medium
  • 148. Sort List - Medium
  • 493. Reverse Pairs - Hard

5. Quick Sort

Core Idea: Choose a 'pivot' element. Partition the array such that all elements less than the pivot are on its left, and all greater are on its right. The pivot is now in its final sorted position. Recursively apply this to the left and right sub-arrays.

Complexity Analysis:

  • Time Complexity: O(n log n) best/average, O(n²) worst case
  • Space Complexity: O(log n) for recursion stack
  • Stability: No

LeetCode Problems:

  • 215. Kth Largest Element in an Array - Medium
  • 912. Sort an Array - Medium
  • 973. K Closest Points to Origin - Medium

6. Recursive Bubble Sort

Core Idea: The same iterative process of Bubble Sort, but implemented using recursion instead of nested loops.

Complexity Analysis:

  • Time Complexity: O(n²)
  • Space Complexity: O(n) for recursion stack

LeetCode Problems:

  • No specific LeetCode problems, but good for understanding recursion
  • Can practice on 912. Sort an Array for learning purposes

7. Recursive Insertion Sort

Core Idea: The same iterative process of Insertion Sort, but implemented using recursion.

Complexity Analysis:

  • Time Complexity: O(n²)
  • Space Complexity: O(n) for recursion stack

LeetCode Problems:

  • 147. Insertion Sort List - Medium
