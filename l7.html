Recursion PatternWise: Get a Strong Hold

What is Recursion?

Recursion is a programming technique where a function calls itself directly or indirectly to solve a problem. The recursive approach breaks down complex problems into smaller, similar subproblems until they become simple enough to solve directly.

Key Components of Recursion

1. Base Case

The condition that stops the recursion. Without a proper base case, recursive functions can run infinitely.

2. Recursive Case

The part where the function calls itself with modified parameters to make progress toward the base case.

How Recursion Works

Recursion uses the call stack to keep track of function calls. Each recursive call pushes a new frame onto the stack until the base case is reached, then the stack unwinds as each call returns its result.

Types of Recursion

1. Direct Recursion

Function calls itself directly.

2. Indirect Recursion

Function A calls function B, which calls function A.

3. Tail Recursion

Recursive call is the last operation in the function. Can be optimized by compilers.

4. Non-Tail Recursion

Recursive call is not the last operation; additional computations are performed after the recursive call.

Subsequences Pattern

What are Subsequences?

A subsequence is a sequence derived from another sequence by deleting zero or more elements without changing the order of remaining elements. Unlike subarrays, subsequences are not contiguous.

Key Characteristics

  • Order is preserved from original sequence
  • Not necessarily contiguous
  • Total subsequences for sequence of length n: 2^n (including empty subsequence)

Recursive Approach for Subsequences

Common Strategy: Pick and Don't Pick

At each element, we have two choices:

  1. Include the current element in subsequence
  2. Exclude the current element from subsequence

Template Code for Generating Subsequences

void generateSubsequences(vector& nums, int index, vector& current, vector>& result) {
    if(index == nums.size()) {
        result.push_back(current);
        return;
    }
    
    // Include current element
    current.push_back(nums[index]);
    generateSubsequences(nums, index + 1, current, result);
    current.pop_back();
    
    // Exclude current element
    generateSubsequences(nums, index + 1, current, result);
}

Applications of Subsequences Pattern

  • Generating all possible subsets
  • Checking if a string is subsequence of another
  • Longest Common Subsequence problems
  • Counting specific types of subsequences

Practice Tips

  • Always identify the base case first
  • Draw recursion tree to visualize the flow
  • Practice both including and excluding element patterns
  • Understand time and space complexity of recursive solutions
  • Learn to convert recursive solutions to iterative ones
