Sliding Window & Two Pointer Techniques

1. Two Pointer Technique

Concept

The Two Pointer technique uses two pointers to traverse data structures (typically arrays or strings) from different positions. The pointers can move in the same direction, opposite directions, or at different speeds.

Common Patterns

  • Opposite Direction: Pointers start at both ends and move toward each other
  • Same Direction: Both pointers start at the beginning and move forward
  • Fast-Slow Pointers: One pointer moves faster than the other

When to Use

  • Sorted array problems
  • Pair sum problems
  • Palindrome validation
  • In-place array modifications
  • Linked list cycles

LeetCode Problems

  • Two Sum II - Input Array Is Sorted
  • 3Sum
  • Container With Most Water
  • Valid Palindrome
  • Remove Duplicates from Sorted Array

2. Sliding Window Technique

Concept

The Sliding Window technique maintains a window (subarray) that slides through the array/string. It's optimized to avoid redundant calculations by adjusting window boundaries.

Key Components

  • Window: Contiguous subarray/substring being considered
  • Window Boundaries: Left and right pointers defining the window
  • Window Condition: Condition that determines when to shrink/expand the window

When to Use

  • Subarray/substring problems with fixed or variable size
  • Problems asking for minimum/maximum subarray satisfying conditions
  • String permutation problems
  • Problems with constraints (sum, unique characters, etc.)

Types of Sliding Windows

  • Fixed Size: Window size is predetermined
  • Variable Size: Window size changes based on conditions

LeetCode Problems

  • Longest Substring Without Repeating Characters
  • Minimum Window Substring
  • Sliding Window Maximum
  • Permutation in String
  • Longest Repeating Character Replacement

Comparison Table

Aspect Two Pointer Sliding Window
Primary Use Pair searching, opposite ends traversal Contiguous subarray/substring problems
Pointer Movement Independent or coordinated movement Sequential sliding movement
Typical Data Structure Arrays, Strings, Linked Lists Arrays, Strings
Complexity Often O(n) Often O(n)
Window Concept No explicit window Explicit window maintained

Implementation Tips

Two Pointer Tips

  • Sort the array first if needed
  • Clearly define pointer movement conditions
  • Handle duplicate values carefully
  • Consider edge cases (empty input, single element)

Sliding Window Tips

  • Use hash maps for character counting
  • Maintain window invariants consistently
  • Track window state efficiently
  • Determine when to shrink/expand the window
